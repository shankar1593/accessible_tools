# AI Transparency

This project was built with significant AI assistance, and that is an explicit part of its story.

## AI systems used

- **Claude (Anthropic)** — major collaboration across architecture, implementation, accessibility remediation, and tests
- **GitHub Copilot in VS Code** — iterative coding support, refactoring, and documentation drafting

## What AI contributed

AI support was used for:

- Architecture planning for separating `calculator.py` (pure logic) from `app.py` (Dash UI)
- Implementation code across worksheets and callbacks
- All 8 WCAG/ARIA fixes introduced during accessibility audit
- Test suite generation and expansion, including printed output helpers and summary-oriented test ergonomics
- User-facing guides and project documentation drafts
- Documentation polishing for consistency and onboarding clarity

## What the human contributor led

Human leadership and expertise defined the project direction:

- Defined the core problem: blind and low-vision candidates preparing for CFA, FRM and other competitive finance exams need accessible BA II Plus-equivalent practice workflows
- Provided domain context and worksheet usage priorities for CFA, FRM and related finance exam preparation
- Gave UI and behavior feedback from screenshot reviews and workflow testing
- Directed design decisions, tradeoffs, and scope boundaries
- Drove debugging decisions using real screen-reader feedback and practical usage goals

## Philosophy

Accessibility tools for blind users should not exist only when a creator happens to have the exact technical background to build them alone. AI can help close implementation gaps faster, but responsibility still sits with humans to define goals, validate outcomes, and engage the affected community.

## Why this is documented openly

This is not something to hide. It is a meaningful and positive part of how this project was built, and transparency helps others reproduce, critique, and improve the process.
